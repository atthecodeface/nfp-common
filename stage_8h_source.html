15  * @file firmware/lib/sync/stage.h
16  * @brief Staged initialization sycnrhonization library
17  *
18  * This is a library of functions used by application firmware to
19  * provide a standard way to order threads and islands
20  *
21  * The macro SYNC_STAGE_SET_PREINIT should be used in the C source
22  * every ME that is to be synchronized. This should be invoked with
23  * the number of stages that are used (in the application), the number
24  * of contexts used in the microengine, the number of microengines
25  * used in the island, and the number of islands used in the
26  * application
27  *
28  * It is then expected that every context in the microengine will
29  * eventually invoke sync_stage_set_stage_complete(NUM_STAGES) to
30  * indicate that it has completed initialization.
31  *
32  * Prior to invoking sync_stage_set_stage_complete with NUM_STAGES,
33  * the thread may invoke it with a smaller number (still greater than
34  * 0). This will permit all threads in the application that wait for
35  * that value to synchronize, permitting initialization to occur in
36  * stages.
37  *
38  * For example, threads may set up queues in a first stage, then wait
39  * for host interaction to start in a second stage, then enter the
40  * main execution phase and invoke the final main application thread
41  * code.
42  *
43  * A thread which only host interaction to start and a main thread may
44  * call sync_stage_set_stage_complete(QUEUE_STAGE), then run its host
45  * interaction, then issue sync_stage_set_stage_complete(ALL_INIT).
46  *
47  * A thread which has queues to initialize and no host interaction may
48  * perform its queue initialization then just issue
49  * sync_stage_set_stage_complete(ALL_INIT).
50  *
51  */
