Detailed Description

Packet generator memory support.

This file provides functions to support loading of the memories in an NFP for the packet generator, from a set of packet generator files created (for example) in pktgen_lib.py.

The set of files includes a schedule, a script, and at least one packet data region.

An instance of the 'pktgen_mem_layout' structure must be allocated, and then filled by loading from a directory.

The layout can then be allocated within the NFP, using an allocation callback and hints for regions of the data to be placed in suitable memories. A single region may be placed in more than one memory of the NFP, being split up in no smaller than 'min_break_size' pieces.

After allocation, the structuce can be patched up - the schedule particularly refers to absolute NFP memory addresses for packets, and the addresses clearly depend on the allocation.

After patching, the layout can be loaded into an NFP, with a callback for each region of memory that needs to be loaded.

Enum regions

A packet generator memory load has one schedule region, one script region (optional), and at least one packet data region. These are loaded from files.

The enumeration here is used in the layout structure and the allocation hints.
